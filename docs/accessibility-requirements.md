# Say What? Accessibility Requirements

## Purpose

This document defines accessibility requirements for Say What?, especially for users who are hearing-impaired or rely on hearing aids, earbuds, or headphones.

## Accessibility Guidelines

- Throughout the app, use straightforward wording.
- When feasible, provide huge text sizes.
- Make sure buttons and tap targets are big enough to be readily pressed.
- Make use of a strong contrast between the background and the text.
- Use more than just music to convey crucial information.
- Clearly display connection status messages visually.
- Provide textual error notifications in addition to audio or warnings.
- Allowing manual entry will make QR scanning unnecessary.
- Make it simple to access Help instructions from the setup and home screens.
- VoiceOver labels for buttons, fields, and images are supported.

## UI Considerations

### Home Screen

- The connection status should be easy to see.
- The Setup, Help, and About buttons should be clearly labeled.
- Text should be readable and not too small.

### Setup Screen

- Clear labels for text boxes, like "Network ID" and "Network Password," are essential.
- The Connect button need to resemble an actual button.
- In the event that QR scanning is unsuccessful, users should be able to manually enter venue details.
- Required fields must to be labeled with clarity.

### QR Scanner Screen

- The app should explain that camera access is needed.
- If camera access is denied, the user should be able to use manual setup.
- The scanner should give a clear visual result when a QR code is detected.

### Help Screen

- Help instructions should be written in short steps.
- Wi-Fi and Bluetooth instructions should be separated clearly.
- The screen should be useful without requiring sound.

### Error Messages

- Error messages should explain what went wrong.
- Error messages should tell the user what to try next.
- Error states should not rely only on color.

## Hearing Accessibility

- Instructions that are solely auditory should not be used by the app.
- Every significant piece of information ought to be displayed both visually and textually.
- The connection status ought to be displayed on the screen.
- Users should be instructed to check wired headphones, Bluetooth earbuds, or hearing aids.

## Visual Accessibility

- The size of the text should allow for comfortable reading.
- Low contrast text should be avoided by the app.It should be simple to recognize buttons and input areas.
- The layout should remain straightforward and uniform on all screens.

## Motor Accessibility

- Buttons should be large enough for users to tap.
- Important actions should not require precise gestures.
- Users should be able to complete setup with normal taps and typing.
