# Say What? Demo Flow

## Purpose

This document defines the exact demo sequence for presenting the Say What? app. The goal is to make the demonstration smooth, repeatable, and easy for any team member to present.

## Demo Goal

Demonstrate the main Say What? user flow from app launch to audio playback, including connection and basic interaction with the audio controls.

## Exact Demo Steps

1. Open the Say What? app.
2. Show the home screen.
3. Navigate to the Audio Control screen.
4. Press Connect.
5. Wait for the app to show the connecting state.
6. Wait for the app to transition to the live state.
7. Press Play or begin the audio demonstration.
8. Confirm that audio can be heard through the intended output device.
9. Adjust the volume if needed.
10. Show that the user can disconnect or stop playback.
11. Return to the previous screen or end the demo cleanly.

## Demo Checklist

- [ ] App launches successfully
- [ ] Home screen appears correctly
- [ ] Audio Control screen opens
- [ ] Connect action works
- [ ] Connecting state is visible
- [ ] Live state is visible
- [ ] Audio playback works
- [ ] Volume can be adjusted if needed
- [ ] Disconnect or stop action works
- [ ] Demo can be completed smoothly without confusion

## Team Demo Readiness

Any team member should be able to complete the demo by following the exact steps above.

## Notes

- The demo should focus on the clearest successful path.
- If there is a known unstable feature, the team should avoid relying on it during the presentation unless it has already been tested successfully.
- The demo presenter should know in advance which device, audio output, and network setup will be used.
